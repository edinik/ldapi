# 增加日间夜间模式切换

## Goal

为 LDAPI 的公开目录、登录页和全部管理页面增加统一的日间/夜间模式切换，让用户能够选择舒适的显示模式，并在后续访问中保持该选择。

## Background

- 项目已使用 shadcn/ui 默认 neutral 语义主题，`src/app/globals.css` 同时具备 `:root` 浅色 token 和 `.dark` 深色 token。
- Tailwind 已配置 `@custom-variant dark (&:is(.dark *))`，现有 shadcn 组件包含暗色状态样式，但根页面目前不会添加 `.dark` class。
- `src/app/layout.tsx` 已在 `<html>` 使用 `suppressHydrationWarning`，仓库尚无主题 Provider、切换组件或偏好持久化逻辑。
- 登录页 Turnstile 当前固定使用浅色主题，需要与页面主题同步。
- 上一次 shadcn 迁移明确保留 `.dark` token 作为后续标准主题能力，不恢复旧暖色 editorial 主题。

## Requirements

### R1 全局主题范围

- 日间和夜间模式必须作用于公开目录、登录页和全部管理页面。
- 页面颜色继续使用现有 shadcn neutral 语义 token，不创建第二套页面颜色体系。
- 浏览器原生控件的 `color-scheme` 必须与已解析主题一致。

### R2 偏好解析与持久化

- 主题偏好仅保存在当前浏览器的 `localStorage`，存储键为 `ldapi-theme`。
- 唯一有效的显式偏好值是 `light` 和 `dark`；缺失值或其他非法值均视为未设置。
- 未设置显式偏好时，主题跟随 `prefers-color-scheme`，并在操作系统主题变化时同步更新。
- 用户首次手动切换后，保存与切换结果一致的明确 `light` 或 `dark` 值；此后操作系统主题变化不得覆盖该显式选择。
- 刷新、客户端导航和重新打开页面后必须保持已保存的显式主题。

### R3 首屏与客户端同步

- 根布局必须在页面内容绘制前解析并应用主题，避免先显示错误主题再切换造成的明显闪烁。
- React 客户端状态必须从根元素的实际主题收敛，不得因服务端未知浏览器偏好而产生 hydration 错误。
- 主题切换后，根元素 class、`color-scheme`、切换控件状态和主题消费者必须同步更新。

### R4 切换控件

- 全站复用一个基于现有 shadcn `Button` 和 lucide 图标的主题切换组件。
- 控件必须支持键盘操作、保留可见焦点、声明 `type="button"`，并向辅助技术公开当前状态和切换目标。
- 控件使用现有页面的页头/操作区：公开页位于“管理入口”旁，登录页位于卡片页头，管理页位于各自页头操作区。
- 不使用可能遮挡移动端内容或 sticky 表单提交栏的全局悬浮按钮。

### R5 第三方控件同步

- 登录页 Turnstile 必须使用当前已解析的 `light` 或 `dark` 主题。
- 主题变化时必须安全重建 Turnstile，并清除旧 widget 对应的客户端 token，避免提交过期或与已卸载 widget 关联的值。

### R6 验证

- 自动化测试覆盖存储值校验、未设置时的系统主题解析，以及日间/夜间的下一状态计算。
- 完成全量类型检查、测试、lint 和生产构建。
- 在桌面和移动端分别检查公开页、登录页及代表性管理页面的浅色/深色显示和控件位置。

### R7 登录页 TOTP 可见性

- 登录页仅在当前管理员已启用 TOTP 时显示验证码输入框。
- TOTP 已启用时验证码输入框必须为必填；未启用时不得显示无来源的验证码字段。
- 后端现有的 TOTP 校验逻辑保持不变，页面不得读取或暴露密钥本身。

## Acceptance Criteria

- [x] AC1：用户可从公开页、登录页及任一管理页面切换日间/夜间模式，页面语义色和 shadcn 组件同步变化。（R1、R4）
- [x] AC2：合法的显式偏好跨刷新、导航和重新打开页面保持；缺失或非法偏好跟随系统主题，并仅在无显式偏好时响应系统变化。（R2）
- [x] AC3：首次绘制前应用正确主题，浏览器控制台没有新增 hydration 错误，页面加载时没有明显错误主题闪烁。（R3）
- [x] AC4：主题按钮可通过键盘操作、焦点可见，并具有正确的当前状态和切换目标无障碍名称。（R4）
- [x] AC5：Turnstile 在两种主题下与页面一致，切换主题后重建 widget 且不会沿用旧 token。（R5）
- [x] AC6：主题纯逻辑测试通过，`npm run check`、`npm run build` 和 `git diff --check` 通过，并完成浅色/深色桌面与移动端检查。（R6）
- [x] AC7：管理员未启用 TOTP 时登录页不显示验证码字段；启用后显示必填字段，且页面只接收布尔状态。（R7）

## Out of Scope

- 自定义品牌主题、独立配色编辑器或多于日间/夜间的主题选项。
- 按账号同步主题、数据库字段、服务端主题 API 或跨浏览器同步。
- 提供“恢复跟随系统”的第三个 UI 选项；未手动选择的用户仍自动跟随系统。
- 修改现有 shadcn neutral token 为旧暖色、珊瑚色或 editorial serif 体系。

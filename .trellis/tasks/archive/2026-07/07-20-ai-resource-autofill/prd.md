# AI 生成资源信息

## Goal

在“添加资源”页面选择“工具项目”时，提供 AI 生成内容能力，减少手工查找和录入资源资料的成本，并允许用户在提交前审阅、选择和修改生成结果。

## Background

- 当前新增资源表单包含类型、标题、简介、标签、GitHub、官网和演示站等字段。
- 表单标签控件已支持复用既有标签、创建自由标签及大小写不敏感去重。
- 项目已有服务端 OpenAI-compatible `/chat/completions` 生成链路、管理端 AI 配置，以及 `web_search` 等研究工具声明。
- 既有 AI 生成交互约定为：生成期间只展示状态，不把未校验的流式半成品写入编辑区；完整结果校验成功后再一次性回填。
- 本功能只面向新增“工具项目”，不在 LinuxDo 教程或资源编辑页提供。

## Requirements

- R1：仅在新增资源且类型为“工具项目”时展示和启用 AI 生成功能。
- R2：标题为必需输入；存在官网或 GitHub 地址时，将其作为优先事实来源，否则按标题研究项目。
- R3：生成一条适合作为公开资源卡片展示的一句话中文简介。
- R4：返回多个适合当前资源的候选标签，并尽量复用系统已有标签。
- R5：识别 GitHub、官网和演示站地址；无法确认时返回空值，不编造链接。
- R6：生成结果只填充当前为空的简介、GitHub、官网和演示站字段，任何已有值不得被 AI 覆盖。
- R7：AI 标签仅作为未选中的候选项展示；用户主动点击后才加入已有标签集合，并沿用现有去重规则。
- R8：生成内容必须先回填到表单供用户审阅，最终仍由用户主动提交资源。
- R9：生成过程必须有明确的加载、成功和失败反馈；失败不能清空或修改已有表单内容。
- R10：AI 请求必须在已认证的服务端接口执行，并沿用后台保存的 AI Base URL、API Key、模型和推理强度配置。
- R11：生成结果必须经过 JSON 结构、简介、标签和 URL 校验后再回填，不能把未完成或格式错误的模型输出写入表单。
- R12：应用服务端不直接抓取用户输入或 AI 返回的任意网址；网页研究交给已配置 AI 服务的研究工具，避免引入 SSRF 通道。

## Acceptance Criteria

- [x] AC1：新增资源选择“工具项目”后可触发 AI 生成；选择“LinuxDo 教程”或进入资源编辑页时不提供该能力。
- [x] AC2：未填写标题时不发送请求，并提示先填写标题。
- [x] AC3：只填写标题也能获得一句话简介和候选标签。
- [x] AC4：填写官网或 GitHub 地址后生成时，请求明确要求 AI 优先使用这些地址研究。
- [x] AC5：完整且校验通过的结果可一次性填充空简介，并展示多个未选中的候选标签。
- [x] AC6：点击候选标签后才加入表单，且不会重复已有标签。
- [x] AC7：系统只回填格式有效的 GitHub、官网和演示站 URL；无法确认或无效的地址不回填。
- [x] AC8：当简介或任一链接字段已有值时，生成不会覆盖该值；空字段可被有效结果填充。
- [x] AC9：用户可在提交前编辑所有 AI 回填内容；AI 生成不会自动提交表单。
- [x] AC10：AI 查询或生成失败时展示可理解的错误信息，并保留用户原有输入。
- [x] AC11：浏览器请求和响应中不包含已保存的 AI API Key；未配置 AI 时返回明确提示。
- [x] AC12：服务端拒绝缺少标题、格式错误或不符合资源建议契约的请求/响应，并保持安全的 HTTP 错误边界。

## Out of Scope

- LinuxDo 教程类型的 AI 内容生成。
- 已发布资源编辑页或批量 AI 补全。
- 自动提交或绕过现有资源校验与审核流程。
- 应用服务器主动抓取、解析或缓存第三方网页内容。
- 新增 AI 配置项、数据库字段或迁移。

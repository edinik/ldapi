# 管理端表单与 CRUD 重构

## Goal

在不改变管理端行为和外部契约的前提下，减少站点、模型、资源维护流程中的重复实现，建立清晰且可测试的表单、客户端请求和服务端写入边界。

## Background

- 三个实体表单重复实现表单分区、文本字段、复选项和提交栏。
- 六个新增/编辑客户端重复维护 JSON 请求、saving 状态和成功跳转。
- 六组实体 Route Handler 方法重复认证和直接执行 Drizzle 写操作。
- 当前全量测试有两条源码正则测试因正常代码演进失效，`npm run lint` 被非产品的 `.pi` 运行时代码阻断。

## Requirements

- 提取稳定重复的表单 UI 原语，但保留模型选择、站点模型价格、资源标签等领域组件。
- 为站点、模型、资源建立各自的纯表单序列化器，并用行为测试锁定现有 JSON 负载。
- 建立小型共享 JSON mutation 能力，统一 pending 和请求结果；实体页面继续配置 URL、method、确认文案和跳转。
- 将站点、模型、资源的 Drizzle 写操作迁入独立服务，服务接受可注入数据库实例以支持内存 SQLite 测试。
- Route Handlers 继续负责认证和 HTTP 响应，保持现有路径、method、状态码、响应字段和错误文案。
- 保持所有表单字段名称、默认值、布局、按钮文案、删除语义和跳转目标不变。
- 用行为或稳定契约测试替代登录、登出相关的源码文本正则断言。
- 为产品代码建立可重复的 `test`、`typecheck`、`lint`、`build` 质量命令；agent/Trellis 运行时目录不纳入产品 lint。
- 不增加新的运行时依赖，不修改数据库 schema 或迁移。

## Acceptance Criteria

- [x] 三个表单不再各自定义重复的 `Section`、`TextField`、复选项组和提交栏结构。
- [x] 站点、模型、资源表单提交负载与重构前一致，并有直接测试覆盖。
- [x] 新增/编辑客户端通过共享 mutation 能力发起请求，实体特有操作仍清晰可读。
- [x] Route Handlers 不再直接包含实体写入查询链，写操作由三个领域服务拥有。
- [x] 内存 SQLite 测试覆盖模型、资源和站点（含站点模型同步）的关键写入行为。
- [x] 登录、登出测试不读取源码做正则匹配；全量测试通过。
- [x] `npm run typecheck`、`npm test`、`npm run lint`、`npm run build` 均成功。
- [ ] 现有管理端手工回归通过，且无 UI、API、跳转或 schema 变化。

## Out of Scope

- 首页目录查询与 DTO 重构，由第二阶段任务负责。
- UI 改版、表单库迁移、API 版本化和数据库迁移。
- 将三个领域强制统一为通用 CRUD repository。
